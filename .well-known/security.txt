# Vulnerability Disclosure Program
# This project follows a coordinated disclosure policy.
# Please report vulnerabilities responsibly.
#
# Response Timeline:
# - Acknowledgment: 24 hours
# - Assessment: 7 days
# - Fix: 30 days (critical), 90 days (non-critical)
# - Public Disclosure: 90 days after fix (coordinated with reporter)
#
# Scope:
# - GraphQL API injection attacks
# - ArangoDB query injection (AQL)
# - XSS in web interface
# - Authentication/authorization bypasses (Phase 2+)
# - Sensitive data exposure
# - Dependency vulnerabilities
#
# Out of Scope:
# - DDoS attacks (no production deployment in Phase 1)
# - Social engineering
# - Physical security
#
# Severity Levels:
# - Critical: RCE, auth bypass (24h response)
# - High: Injection, XSS, privilege escalation (7 days)
# - Medium: CSRF, information disclosure (30 days)
# - Low: Minor info leaks (90 days)
#
# Bug Bounty: No formal program yet (Phase 1 PoC)
# Recognition: Security researchers credited in CHANGELOG.md and humans.txt
#
# Security Features:
# - Type Safety: Elixir compile-time checks, GraphQL schema
# - Memory Safety: BEAM VM isolation
# - Input Validation: Ecto changesets, parameterized queries
# - EU GDPR Compliance: Anonymization, right to erasure
# - Audit Logging: All mutations logged
# - HTTPS/TLS: Required in production (Phase 2)
# - Dependency Scanning: Automated via CI/CD
#
# Compliance:
# - OWASP Top 10: Prevention guidelines followed
# - EU GDPR: Article 25 (Privacy by Design), Article 32 (Security)
# - ISO 27001: Aspiring (future certification)
#
# Infrastructure (Phase 2):
# - Hosting: Hetzner Cloud (EU data sovereignty)
# - Database: ArangoDB Oasis (managed, encrypted)
# - SSL: Let's Encrypt with HSTS, TLS 1.3
# - Firewall: Ports 80/443 only
#
